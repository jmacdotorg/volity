Vocabulary

There is a lot of Volity jargon sprinkled through the wiki, and also in the client interface, in source code, etc. We would like all of this jargon to be consistent and (preferably) not confusing.

This page tracks confusing terms. Where possible, we should change other wiki pages, UI elements, web site elements, and so on to match what's defined here.

User

A Volity user is anyone who has used the system.

In the context of a table, everyone present at the table (except the referee) is a user. A user can be an observer (standing, not taking part in the game) or a player (seated, taking part in the game).

Outside the context of a table, "player" and "user" mean the same thing. (Everyone using Volity is a game-player of some sort.) But "user" is preferable. Reserve "player" for someone who is playing a particular game at the moment.

A bot is a user, albeit a cybernetic user.

Table

A Volity table is a MUC with a referee present.

It is possible to join a table with a plain Jabber client (like iChat). You will see all the users, and you can take part in table chat, but you won't see any game moves.

Referee

From the player's point of view, a referee is the Volity entity which is present at the table, which mysteriously manages the game. (You don't really see it do any of this game management, but it's doing it. )

From the developer's point of view, the referee is divided into two parts, called (confusingly) the referee object and the game object. The referee object handles all the communication and the general Volity bookkeeping. The game object handles all the game logic and the game-specific decisions.
