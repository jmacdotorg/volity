Gamut

Volity Gamut is the "official" Volity client, with a pretty Java-based GUI. It is capable of fetching and loading UI files written in SVG and ECMAScript.

Gamut's homepage is at http://www.volity.org/projects/gamut/ .

The principal developer is currently Zarf. The original developers were Karl and Doug, with jmac wandering through occasionally.

Want to help localize Gamut? We are seeking volunteers to help translate the client interface into languages other than English.

(Gamut was previously called "Javolin", which is why the module name and the Java code say "Javolin" all over.)

Libraries

Gamut uses the following third-party libraries:

  • [Smack], an XMPP (Jabber) library from Jive Software.
  • [Rhino], an ECMAScript library from Mozilla
  • [Batik], a SVG library from Apache
  • [Flying Saucer], an XHTML library
  • [MP3SI], an MP3 library

Running Gamut

See Gamut Quick Start for information on obtaining and running the current Gamut release.

The primary download sites are:

  • [the latest official release]
  • [the latest nightly build]

You can also get the latest bleeding-edge code through Volity SVN, by doing a checkout of the Javolin module. Included in the software is a ReadMe_Build file that will guide you through the process of building your own copy of Gamut, inlcuding URLs for the specific versions of the third-party libraries you'll need to download.

Once you have it running, you can try connecting to one of the test servers and watch as it automagically downloads the right SVG UI files and plays games!
